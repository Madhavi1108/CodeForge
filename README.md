# CodeForge

CodeForge is a distributed code execution platform with a React editor, FastAPI API, Kafka-backed workers, sandboxed Docker execution, credits, authentication, and monitoring.

## What is included

- FastAPI backend with JWT authentication, job submission, billing credits, and Prometheus metrics.
- React + TypeScript frontend built with Vite, Monaco Editor, Zustand, Axios, and Lucide icons.
- Kafka worker that executes submitted code in isolated Docker containers.
- AI worker queue hooks for explanation and plagiarism jobs.
- PostgreSQL schema bootstrap, Redis, Kafka, Prometheus, and Grafana via Docker Compose.

## Prerequisites

- Docker and Docker Compose
- Node.js and npm, for running the frontend locally
- Python 3.11+, if running scripts outside Docker

The worker uses the host Docker socket to create sandbox containers. Make sure Docker is running before starting the stack.

## Quick start

Start the backend services, workers, databases, and monitoring:

```bash
docker compose up --build
```

In another terminal, start the frontend:

```bash
cd frontend
npm install
npm run dev
```

Open the Vite URL, usually:

```text
http://localhost:5173
```

The API is available at:

```text
http://localhost:8000
```

## 🔐 Default Login

On startup, the API creates a default admin user if it does not already exist:

```text
Email: engineer@codeforge.ai
Password: password
```

Newly registered users receive 100 credits. The default admin receives 1000 credits.

## API overview

Authentication:

- `POST /auth/register` creates a user.
- `POST /auth/token` returns a bearer token.

Code execution:

- `POST /jobs/` submits a code execution job.
- `GET /jobs/{job_id}` returns job status and result.
- `GET /jobs/failed` lists failed jobs for the current user.

Credits and AI:

- `GET /credits/` returns the current credit balance.
- `POST /credits/topup` adds credits to the current account.
- `POST /ai/explain` queues a code explanation job.
- `POST /ai/plagiarism` queues a plagiarism check job.

Supported execution languages are `python`, `cpp`, and `java`.

## Smoke test

With the Docker stack running, execute:

```bash
python test_all.py
```

The script logs in with the default account, submits a Python job, polls for completion, and checks the AI queue endpoints.

## Development notes

- The frontend dev server proxies `/api/*` requests to `http://localhost:8000`.
- The backend reads configuration from environment variables and can also load a `.env` file.
- PostgreSQL tables are initialized from `backend/db/init.sql`.
- The execution worker pulls sandbox runtime images on demand:
  - `python:3.11-slim`
  - `gcc:12-bullseye`
  - `eclipse-temurin:17-jdk-jammy`
- Jobs are idempotent per user through the submitted `idempotency_key`.
- The worker retries system failures before publishing to the `dead_letter_queue`.

## Useful commands

```bash
# Start the full stack
docker compose up --build

# Stop services
docker compose down

# Stop services and remove database volume
docker compose down -v

# Build the frontend
cd frontend
npm run build

# Lint the frontend
cd frontend
npm run lint
```

## Configuration

Common backend and worker environment variables:

| Variable | Default |
| --- | --- |
| `DATABASE_URL` | `postgresql://root:rootpassword@localhost:5432/codeforge` |
| `REDIS_URL` | `redis://localhost:6379/0` |
| `KAFKA_BOOTSTRAP_SERVERS` | `localhost:9092` |
| `JWT_SECRET` | `super_secret_key` |
| `CORS_ORIGINS` | `http://localhost:5173` |
| `RATE_LIMIT_PER_MINUTE` | `100` |

For production-like deployments, replace the default secrets and database credentials before exposing the services.
